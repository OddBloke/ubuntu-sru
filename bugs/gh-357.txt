[Impact]
When a user is configuring apt, we must ensure the provided yaml conforms to
our expected schema.

```
cat > setup_proposed.sh <<EOF
#/bin/bash
mirror=http://archive.ubuntu.com/ubuntu
echo deb \$mirror \$(lsb_release -sc)-proposed main | tee /etc/apt/sources.list.d/proposed.list
apt-get update -q;
apt-get install -qy cloud-init;
EOF

for SERIES in bionic eoan focal xenial; do
   echo '=== BEGIN ' $SERIES
   ref=$SERIES-proposed
   name=test-$SERIES
   lxc delete $name --force 2> /dev/null
   lxc launch ubuntu-daily:$SERIES $name

   lxc file push valid_apt.yaml $name/var/tmp/
   lxc file push invalid_apt.yaml $name/var/tmp/

   lxc exec $name -- cloud-init status --wait --long
   echo 'Expecting no failures on invalid schema pre-upgrade because no schema apt defined'
   lxc exec $name -- cloud-init devel schema -c /var/tmp/invalid_apt.yaml || echo "FAILURE"
   lxc exec $name -- cloud-init devel schema -c /var/tmp/valid_apt.yaml || echo "FAILURE"

   # upgrade cloud-init to proposed
   lxc file push setup_proposed.sh $name/var/tmp/
   lxc exec $name -- bash /var/tmp/setup_proposed.sh
   # re test on current cloud-init that both invalid and valid_apt.yaml pass (because we have no schema validation
   echo "Expect schema failure on invalid_apt"
   if [ "$SERIES" = "xenial" ]; then
       echo "Xenial does not have jsonschema dependecy, making this type of checking to not work on it"

       success_msg="SUCCESS: xenial allowed invalid schema because it lacks python3-jsonschema dep"
       failure_msg="FAILURE: Xenial failed invalid_schema which it should have passed"
       lxc exec $name -- cloud-init devel schema -c /var/tmp/invalid_apt.yaml && echo $success_msg || echo $failure_msg
   else
       lxc exec $name -- cloud-init devel schema -c /var/tmp/invalid_apt.yaml || echo "SUCCESS: $SERIES rejected invalid schema"
   fi
   echo "Expect success on valid_apt.yaml"
   lxc exec $name -- cloud-init devel schema -c /var/tmp/valid_apt.yaml || echo "FAILURE: $SERIES should have accepted the valid schema"
done
```

[Regression Potential]

Some users may rely on apt config that do not follow the enforced pattern in this schema.
However, as far as I can see, this can be mitigated by the users adapting their apt configuration
to follow that pattern. Because this can be easily solvable, we believe this is an acceptable risk to take.
